# 文件上传功能 - 快速启动指南

## 🚀 5分钟快速开始

### 第一步：创建数据库表

在MySQL数据库中执行以下SQL：

```sql
CREATE TABLE `attachment` (
  `attachment_id` INT PRIMARY KEY AUTO_INCREMENT COMMENT '附件ID',
  `file_name` VARCHAR(255) NOT NULL COMMENT '文件名',
  `file_path` VARCHAR(500) NOT NULL COMMENT '文件路径',
  `file_type` VARCHAR(100) COMMENT '文件类型',
  `file_size` BIGINT COMMENT '文件大小（字节）',
  `task_id` INT COMMENT '关联的任务ID',
  `user_id` INT NOT NULL COMMENT '上传用户ID',
  `stage_id` INT NOT NULL COMMENT '关联的阶段ID',
  INDEX idx_user_id (`user_id`),
  INDEX idx_stage_id (`stage_id`),
  INDEX idx_task_id (`task_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='附件表';
```

### 第二步：检查配置文件

确认 `src/main/resources/application.properties` 包含以下配置：

```properties
# 文件上传配置
spring.servlet.multipart.max-file-size=10MB
spring.servlet.multipart.max-request-size=10MB
upload.filePath=D:/project_files/hanjia
```

> 💡 可根据需要修改 `upload.filePath` 为其他路径

### 第三步：启动应用

```bash
mvn spring-boot:run
```

或者在IDE中直接运行 `DemoApplication.java`

### 第四步：测试功能

#### 方式一：使用可视化测试页面

1. 打开浏览器访问：`http://localhost:8888/file-upload-test.html`
2. 输入Token（如果有认证）
3. 选择阶段ID
4. 选择文件并上传
5. 查看附件列表

#### 方式二：使用cURL命令

**上传文件**:
```bash
curl -X POST http://localhost:8888/api/file/upload \
  -F "file=@test.pdf" \
  -F "stageId=1"
```

**获取附件列表**:
```bash
curl -X GET http://localhost:8888/api/file/list?stageId=1
```

**删除文件**:
```bash
curl -X DELETE http://localhost:8888/api/file/delete/1
```

#### 方式三：使用Postman

1. **上传文件**
   - Method: POST
   - URL: `http://localhost:8888/api/file/upload`
   - Body (form-data):
     - file: [选择文件]
     - stageId: 1

2. **获取列表**
   - Method: GET
   - URL: `http://localhost:8888/api/file/list?stageId=1`

3. **删除文件**
   - Method: DELETE
   - URL: `http://localhost:8888/api/file/delete/1`

## 🔍 常见问题排查

### 问题1：启动失败

**可能原因**: 
- 端口8888被占用
- 数据库连接失败

**解决方案**:
```bash
# 检查端口占用
netstat -ano | findstr :8888

# 修改端口（在application.properties中）
server.port=8080
```

### 问题2：上传失败 - "未登录或Token无效"

**原因**: 请求缺少有效的Token

**解决方案**:
- 如果项目启用了认证拦截器，需要先登录获取Token
- 在请求头中添加：`Authorization: Bearer YOUR_TOKEN`
- 或者临时禁用拦截器进行测试

### 问题3：上传失败 - "创建上传目录失败"

**原因**: 没有权限创建目录

**解决方案**:
```bash
# Windows: 手动创建目录
mkdir D:\project_files\hanjia

# 或者修改配置到其他有权限的目录
upload.filePath=C:/temp/uploads
```

### 问题4：找不到AttachmentMapper

**原因**: MyBatis扫描路径配置问题

**解决方案**:
确认 `DemoApplication.java` 中有：
```java
@MapperScan(basePackages = {"com.txy.demo.mapper", "com.txy.hanjiapeixun.mapper"})
```

### 问题5：文件上传成功但数据库中没有记录

**原因**: 可能是事务问题或Service注入失败

**解决方案**:
- 检查控制台日志
- 确认AttachmentService已正确注入
- 确认数据库连接正常
